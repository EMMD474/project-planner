# Features

This document details the core features of the **Project Planner** application, focusing on bridging technical workflows with project management.

## 🔗 Git Integration

The Project Planner is designed to be deeply integrated with your Git provider (GitHub, GitLab, Bitbucket, etc.).

- **Branch-Linkage**: Automatically associate Git branches with specific project tasks or milestones.
- **Commit Tracking**: Link commit messages (e.g., using `#task-id`) to automatically update task progress and status.
- **PR Visibility**: View the status of Pull Requests directly within the planner's interface, allowing non-technical stakeholders to see "work in progress" without leaving the tool.
- **Auto-Deployment Status**: Integration with CI/CD pipelines to show when a feature is "Live" or "In Staging".

## 📊 Project Tracking Tools

Modern tools tailored for transparency and efficiency.

- **Interactive Roadmap**: A high-level timeline view for planning quarterly or yearly milestones.
- **Dynamic Kanban Boards**: Drag-and-drop task management for agile teams.
- **Granular Progress Bars**: Visual indicators of task and project completion based on sub-tasks and Git activity.
- **Stakeholder Analytics**: Automated reports showing velocity, bottleneck identification, and resource allocation.

## 🤝 Collaboration & Roles

A unified space for the whole team.

- **Technical View**: Focuses on code links, PR status, and technical blockers.
- **Business View**: Focuses on timelines, budget, and business-level milestones.
- **Contextual Comments**: Threaded discussions directly on tasks, supporting markdown and code snippets.
- **Notification Engine**: Stay updated on task changes, PR approvals, and milestone achievements via email or Slack.

## 🔮 Future Roadmap

- **AI-Assisted Planning**: Automated task breakdown and estimation based on historical data.
- **Dependency Mapping**: Visualise and manage complex inter-task dependencies.
- **Time Tracking**: Integrated time logs for project billing and analysis.
